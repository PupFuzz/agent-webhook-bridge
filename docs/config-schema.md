# Config schema (current-state reference)

The **what**: every configuration key the bridge reads, its type, default, and whether a bad value **fails closed** (throws/5xx/non-zero) or **warns**. The **why** of each choice lives in [`../CLAUDE_DECISIONS.md`](../CLAUDE_DECISIONS.md) (the `DL-NNN` log); this file is the flat reference so you don't have to reconstruct the schema by reading DL-002 + 005 + 006 + 007 + 008 + 010 + 011 + 014 as deltas. (B-11.)

There are three configuration surfaces:

1. **Per-install** — Laravel `.env` → `config/bridge.php`. Same for every agent on one install.
2. **Per-agent** — `<config_dir>/<agent>.yml`. One file per agent; the **filename is the agent name**.
3. **Secrets / state** — files under `<secret_dir>` / `<state_dir>`, not committed.

---

## 1. Per-install (`.env` / `config/bridge.php`)

| Env var | Type | Default | Notes / fail mode |
|---|---|---|---|
| `BRIDGE_DIR` | abs path | — | Base dir for per-agent YAMLs, `shared-identities.json`, secrets, tokens, and `state/`. `config_dir` + `secret_dir` default to it. |
| `BRIDGE_CONFIG_DIR` | abs path | `BRIDGE_DIR` | Override only when config lives elsewhere. Missing/not-a-dir → `bridge:check` **fails**; dispatch reads it per request. Group/world-accessible → **warn** (DL-014). |
| `BRIDGE_SECRET_DIR` | abs path | `BRIDGE_DIR` | Holds HMAC secrets + tokens. Not set / not absolute → `bridge:check` **fails**. Group/world-accessible → **warn** (DL-014). |
| `BRIDGE_INSTALL_SUFFIX` | string | `''` | `-prod` / `-dev` crosstalk marker. `InstallGuard` requires the DB name to **contain** the marker with `-`→`_` (i.e. `_prod` / `_dev`, as a substring) — else the receiver **5xx**s and `bridge:check` **fails** (DL-001). |
| `BRIDGE_RECEIVER_BASE_URL` | http(s) URL | — | This bridge's public webhook URL (used by `bridge:provision`). Malformed → `bridge:check` **fails**. |
| `BRIDGE_KANBAN_API_BASE_URL` | http(s) URL | — | kanban API base. Malformed → `bridge:check` **fails**. |
| `BRIDGE_GITHUB_API_BASE_URL` | http(s) URL | `https://api.github.com` | Only relevant if a github adapter calls the API. |
| `BRIDGE_GITHUB_TOKEN_PATH` | abs path | — | `bridge:reconcile` GitHub read token (DL-184). Authoritative when set: a missing/blank/insecure file **fails loud** with no store/env fallback. Absent → the conventional `<secret_dir>/github/token`, then store-native, then `GH_TOKEN`. |
| `BRIDGE_GITHUB_CREDENTIAL_HELPER` | helper name \| abs path \| `''` | `git-credential-coord` | `bridge:reconcile` store-native token leg (DL-185): resolves a **per-repo** PAT from the coordination store's `[git-credential-map]` via this helper (git wire-format). Bare name → PATH-resolved; a `/`-path used as-is; empty **disables** the store leg. Absent helper → falls through to `GH_TOKEN`. Needs `HOME`/`COORD_CREDENTIALS` in the reconcile env to locate the store. |
| `BRIDGE_STATE_DIR` | abs path | `<config_dir>/state` | `inbox*.jsonl` / seen cursors / handler logs. Point OUTSIDE the 0700 config dir for cross-user inbox reads. |
| `BRIDGE_INBOX_LAYOUT` | `shared` \| `per-agent` \| `both` | `shared` | Where staged lines land. Invalid → `bridge:check` + dispatch **fail closed**. |
| `BRIDGE_DEFAULT_AGENT` | agent name | — | Bare `bridge:inbox` surfaces this agent. Unknown name → `bridge:check` **warns**. |
| `BRIDGE_INBOX_FILE_MODE` | octal string | `0640` | Mode applied to per-agent **inbox** files (cross-user read). Seen-cursor files stay install-user-owned (default umask). |
| `BRIDGE_INBOX_GROUP` | group name | — | Group on per-agent inbox files. **Requires `per-agent` layout** (else `bridge:check` + dispatch **fail closed** — a group-readable shared inbox would leak every agent's intents, DL-006). |
| `BRIDGE_MAX_BODY_BYTES` | int | `262144` (256 KB) | Envelope size cap; oversize → `413` before HMAC work. |
| `BRIDGE_RETENTION_ENABLED` | bool | **`true`** | Prune the append-only stores after each delivery (DL-199), from a terminating callback so the response is already sent. **On by default — an upgrade starts pruning without operator action** (a default nobody sets is why DL-012's cron never ran). `false` ⇒ nothing prunes unless you schedule `bridge:prune`. `bridge:check` **warns** when disabled. |
| `BRIDGE_RETENTION_INTERVAL` | int (seconds) | `86400` | Gap between passes **once the store is drained**. While a backlog remains the gate keeps draining on successive deliveries instead of waiting this out (each pass is still capped by `batch`). `< 1` ⇒ prunes **nothing** + `bridge:check` warns. |
| `BRIDGE_RETENTION_OLDER_THAN` | `Nd` \| `N` \| `''` | `30d` | Delete `webhook_events` (cascading `agent_dispatches`) + trim `inbox*.jsonl` older than this. Same 1–36500 guard as `--older-than`. Unparseable ⇒ prunes **nothing** (never falls back to a default cutoff — that would delete on a typo) + `bridge:check` warns. Empty ⇒ leg off. |
| `BRIDGE_RETENTION_NULL_PAYLOADS_OLDER_THAN` | `Nd` \| `N` \| `''` | `''` (**leg off**) | Null `webhook_events.payload` past the replay window, keeping the row (its `delivery_id` dedup gate + audit metadata). Use a value **smaller** than `older_than`. An optional space optimization, not part of the growth fix — hence off by default. |
| `BRIDGE_RETENTION_BATCH` | int | `500` | Max rows one pass touches **per leg** — the bound that keeps a pass to milliseconds of FPM-worker time instead of a 20k-row `DELETE`. Deleted oldest-first. `< 1` ⇒ prunes **nothing** + `bridge:check` warns. Does **not** bound the inbox trim (a jsonl rewrite is whole-file; the cost is paid once). |
| `BRIDGE_COORD_CONFIG_PATH` | abs path | falls back to ambient `COORD_CONFIG` | Per-install override for the coordination config path. Two installs on one host (`-prod` / `-dev`) share one ambient `$COORD_CONFIG` — set this in that install's `.env` to point them at different coordination projects. **This leg is the cache-safe one**: it resolves through `config/bridge.php`, so it survives `php artisan optimize`. When unset, `bridge:check` falls back to the ambient `$COORD_CONFIG` read live via `getenv()` (see below). Same CLI-only/never-fail semantics as `COORD_CONFIG`. |
| `COORD_CONFIG` | abs path | — | Fallback for the above: the coordination project's `coordination.config.json`. **Deliberately NOT resolved through `config/bridge.php`** — it is read at the `bridge:check` read-site (see below). **Read ONLY by `bridge:check`**, for the DL-200 `move_coord_cards` cross-config terminal compare (resolves that board's `terminal_columns` and compares against `coord_card_terminal_stage_id`). Note this is the **only** non-`BRIDGE_*` env the bridge reads — the name is the framework's, deliberately reused so an operator's existing shell already sets it. **CLI-only by design:** `bridge:check` runs with the operator's environment; the receiver runs under PHP-FPM, which does **not** carry it — so nothing on the request path may read it (a request path bound to an absent file fails silently in the one process nobody watches). **Read live via `getenv()` at the check, NOT baked into `config/bridge.php`** — `php artisan optimize` (the documented deploy step) caches config and would freeze this to whatever the *deploying* shell had, usually nothing, making the compare report CANNOT-VERIFY forever. If you cache config AND want a fixed path regardless of the invoking shell, set `BRIDGE_COORD_CONFIG_PATH` in `.env` (that leg IS cache-safe). Unset/absent/malformed ⇒ the compare reports **CANNOT-VERIFY** (never agreement); it **never fails** `bridge:check`. |
| `BRIDGE_SPAWN_ENABLED` | bool | `false` | Registers the `spawn_detached` handler. Off ⇒ a `spawn_detached` target is a best-effort note, never an execution (DL-011). |
| `BRIDGE_SPAWN_ALLOWLIST` | comma-sep abs paths | `''` | `cmd[0]` must be in it. Empty ⇒ nothing runs. **Allowlist fixed-purpose wrappers, never interpreters** (DL-011). |
| `BRIDGE_SPAWN_SETSID_PATH` | abs path | _(auto)_ | Absolute path to the `setsid` launcher. Unset ⇒ auto-detect (`/usr/bin/setsid`, `/bin/setsid`). Pinned absolute so a payload `env` PATH can't redirect which `setsid` runs (DL-037); fail-closed if none is found. |
| `BRIDGE_CHANNEL_ALLOWED_SOCKET_DIR` | abs path | — | Prefix a **classifier-supplied** `channel_push` socket must sit under; unset ⇒ classifier sockets **refused** (DL-014). Agent-config sockets exempt. |

DB connection (`DB_CONNECTION` / `DB_*`) and `APP_*` are standard Laravel. The DB password lives in `.env` (`DB_PASSWORD`), never in the YAMLs.

## 2. Per-agent YAML (`<config_dir>/<agent>.yml`)

The **filename** (minus `.yml`) is the agent's canonical name and its echo "self" — there is **no `identity.self`** (DL-007). Malformed YAML throws at load → **5xx** (fail-closed; kanban-board holds + redelivers). `chmod 600` the file.

### `identity:` (optional mapping) — the agent's own immutable upstream ids
| Key | Type | Default | Notes |
|---|---|---|---|
| `kanban_user_id` | int | `null` | The agent's kanban `user_id`. Auto-seeded into self echo-suppression. |
| `github_user_id` | int | `null` | The agent's immutable GitHub `sender.id`. Auto-seeded into self echo. |
| `github_login` | string | `null` | **Display-only label** — never a matching key (renameable; DL-002). A stale value logs a one-line drift warning. |

Recognition keys on the numeric ids only; the agent never hand-lists its own id for echo (auto-seeded, DL-007). Modelled by `IdentityConfig` (DL-017).

### `subscriptions:` (expected list — an agent with none receives nothing) — which provider scopes this agent receives
Each entry:
| Key | Type | Default | Notes |
|---|---|---|---|
| `provider` | `kanban` \| `github` | — | Must have an adapter (`bridge:check` **fails** on an unknown provider, DL-015). |
| `scopes` | list | — | kanban board ids (`[5]`) / GitHub `org/repo` slugs. A scope failing `ScopeId` validation is rejected (it's the path-traversal boundary). |
| `event_filter` | list | `[]` (all) | e.g. `["task.*","comment.*"]`. |

Non-array `subscriptions`, or an entry that isn't a mapping, throws at load. Only `kanban` is API-provisionable (`bridge:provision` skips github with a non-zero exit).

### `echo_suppression:` (optional) — OTHER agents only (self is auto-seeded)
| Key | Type | Default | Notes |
|---|---|---|---|
| `treat_as_echo` | list of agent names | `[]` | Other agents' writes to silently skip (by name). |
| `treat_as_echo_ids` | list of raw ids | `[]` | Extra raw provider ids to skip (rarely needed; self ids are auto-seeded). |
| `treat_as_signal` | list of agent names | `[]` | If non-empty, **only** these agents' writes are signal. **Fail-closed:** a name with no matching `<name>.yml` throws at **preflight (`bridge:check`) / first dispatch** (5xx), not at YAML load (DL-007 — `SignalAllowlist`). |

### `classifier:` (optional)
| Key | Type | Default | Notes |
|---|---|---|---|
| `class` | FQCN | `App\Bridge\Classifiers\InboxOnlyClassifier` | Must be resolvable + implement `Classifier`; `bridge:check` resolves it so a typo **fails** at preflight instead of a dispatch-time 5xx. |
| `config` | mapping | `{}` | Typed parameters for a **config-driven classifier** (`ClassifierConfig`), so a shared reference classifier is parameterized without hardcoding per-project specifics. Absent ⇒ all-defaults (back-compat). A present non-mapping — or a malformed key below — **throws** (fail-closed, like the rest of the config). Read by a classifier via `$ctx->agent->classifierConfig`. |

#### `classifier.config:` keys
| Key | Type | Default | Notes |
|---|---|---|---|
| `scope_author_map` | mapping | `{}` | `scope_id` (repo) ⇒ the sole author-agent on that repo — the primary attribution path for a **label-less** impl event (falls back to the `from:`/`FROM:` line). Keys lowercased for case-insensitive scope matching; a non-string key/value **throws**. |
| `families` | list<string> | `[]` (classifier default) | The event families a config-driven classifier runs (its config-gated pipeline). Lowercased; a blank entry **throws**. Recognized by `CoordinationClassifier`: `coord-message` (GitHub coordination messages), `impl-ci-wake` (push→release-branch + CI wake), `kanban-triage` (wake the triage owner on a human-filed untriaged card — DL-168). An unknown family is ignored (forward-compat). Empty ⇒ the classifier's own default (`CoordinationClassifier` → `[coord-message]`; the `KanbanTriageClassifier` shim → `[kanban-triage]`). |
| `impl_repos` | list<string> | `[]` (all subscribed) | **`impl-ci-wake` family.** Gate the impl wake (push→release-branch / `workflow_run` CI) to a repo subset — the family fires only when the event's `scope_id` is in this list. Lowercased for case-insensitive matching. Empty/absent ⇒ every subscribed repo wakes (back-compat). A PM subscribed to both a coord repo and impl repos sets this to its impl repos so a coord-repo push/CI event doesn't self-wake. |
| `drop_title_all_of` | list<list<string>> | `[]` | **`coord-message` family.** Drop (no intent, no wake — before the recipient gate) a coordination subject whose **title** contains **every** substring of any one group (AND within a group, OR across groups) — for bookkeeping-title noise (e.g. `[["Rule E back-merge sync","paper-trail anchor"]]`). Substrings lowercased for case-insensitive matching; a non-list group or an empty substring **throws** (an empty group is ignored). **Blast radius:** for an `issue_comment` the title is the *parent issue's* title, so a matched group also suppresses every comment on that issue — intended for a pure paper-trail anchor (all its activity is noise), but keep groups specific enough not to swallow a live-discussion thread. Empty/absent ⇒ no subject dropped. |
| `wake_membership` | list<string> | `[to_me, to_all, comment_to]` | **`coord-message` family (DL-190).** Which classes grant coord-message **live-wake** (channel_push): `to_me` (a `to:<me>` label), `to_all` (`to:all`), `from_me` (`from:<me>` — activity on a thread you opened), and `comment_to` (DL-192 — a body-`TO:<self>` grant, see below). **DEFAULT `[to_me, to_all, comment_to]`** — `comment_to` is **default-ON since DL-213** (the post-a-reply-and-wait flow — post to a counterparty, wait for their directed reply — is otherwise dark out of the box). **`from_me` stays OUT of the default** — it is the over-wake/self-wake knob: a coordinator opening many threads would else wake on every reply to them. Add `from_me` to also live-wake on **all** activity on threads you opened (a `from:<me>` issue label) — this is **operationally required, not optional, for a PM/coordinator seat** that opens briefs labeled `to:<impl>` (never `to:<self>`) and must live-wake on the impl replies to them; without it those brief-reply wakes downgrade to SessionStart-only pickup. Note: a comment's direction lives in its **body `TO:` line**, not as a `to:<me>` issue label. The body-`TO:` check is three-state (`RecipientAddressing::addresses`): a comment addressed to **someone else** always *narrows* (denies membership, unconditional); a comment with **no** `TO:` line falls back to label membership; and a comment addressed **TO you** grants membership **when `comment_to` is set** (default ON since DL-213). So only if you set `wake_membership` explicitly and OMIT `comment_to` does a directed reply on your own thread shift from live-wake to SessionStart (`from:<self>` / `protocol:invalid`) boundary pickup — `bridge:check` warns on exactly that explicit-narrow config. **`comment_to` (DL-192)** is precisely that grant: it live-wakes on a comment whose body says `TO: <self>` **even when the thread labels don't address you** — closing the cross-thread pull-in gap (a loop-in on a thread you neither opened nor were labelled on) that no label-class can express, since the `to:` label freezes at thread-open. It grants only (and keys on the directed `TO:` line, not authorship, so it is echo-safe under a shared account); the narrow (`TO:` someone-else) and the no-`TO:` label-fallback are unchanged, so `comment_to` is purely additive. This gates membership only; inbox-staging is unchanged. Values lowercased. **UPGRADING:** an install with **no explicit** `wake_membership` will start live-waking on comment-directed replies after DL-213; set `wake_membership` explicitly (omitting `comment_to`) to keep them dark. |
| `impl_non_wake_disposition` | string | `drop` | **`impl-ci-wake` family (DL-190).** Disposition of a **non-wake** terminal impl event (a benign completed `workflow_run`, a non-release-branch push). `drop` (default) = gate-drop, no intent (lean inbox — the pre-DL-190 behavior). `inbox_stage` = build a normal inbox `Intent` (`impl_push` / `impl_ci`) with **no** `channel_push`, keeping a broad CI/push SessionStart history; a channel whose `route_intents` owns waking then wakes on it. A non-terminal `workflow_run` / a branch-delete push is never staged. Any other value falls back to `drop`. |
| `ci_failure_workflow_patterns` | list<string> | `[]` (any workflow) | **`impl-ci-wake` family (DL-197).** Optional workflow-NAME allow-list on the **CI-FAILURE** wake: when non-empty, a failing `workflow_run` wakes **only** if its `name` contains one of these (case-insensitive substring); a failure of any other workflow becomes a non-wake run (dropped, or inbox-staged per `impl_non_wake_disposition`). It narrows **which workflows**, never **which conclusions** — a matched workflow still wakes fail-loud on *any* non-benign conclusion (incl. unknown/future ones). A `workflow_run` with no `name` is never filtered (wakes fail-loud). Values lowercased; a blank entry **throws**. Empty/absent ⇒ every failing workflow wakes (back-compat). **⚠ This inverts the family's fail-loud default: a pattern matching NO real workflow — a typo, or a workflow later renamed — SILENTLY blackholes every CI-failure wake for the scope (no wake, no trace under the default `drop`). Config validation cannot catch a well-formed-but-stale pattern; `bridge:check` surfaces the configured patterns at preflight — verify them against your actual workflow `name:` values.** Orthogonal to `provenance_patterns` (the success-cue path is unaffected). |
| `coord_extra_actions` | mapping<string,list<string>> | `{}` | **`coord-message` family (DL-190).** Extends the fail-safe `subject()` action allow-list per event prefix — `{ pull_request: [synchronize] }` surfaces `pull_request.synchronize` (a PR update) beyond the default (`issues`: opened/reopened; `issue_comment`: created; `pull_request`: opened/reopened/ready_for_review). Keys are the event prefix without the trailing dot; keys+values lowercased; a non-list value or an empty string **throws**. An **unlisted** action still returns null (allow-list, not deny-list — a new GitHub action never auto-surfaces). |

> Family-specific config (e.g. an `impl-ci-wake` family's `benign_conclusions` / CI-name patterns / `release_branch` / `impl_repos`) is read through `ClassifierConfig`'s generic typed accessors (`strings()` / `string()` / `section()` / `stringGroups()`), so a new family adds keys here **without** a schema/contract change.

### `channel:` (optional) — where `channel_push` / `route_intents` delivers
| Key | Type | Default | Notes |
|---|---|---|---|
| `socket` | abs path | `null` | Local UDS. Absolute, no `..`, no null byte (else throws). **Mutually exclusive** with `url`. **`${XDG_RUNTIME_DIR}` / `${uid}` expand at load (DL-039)** so you can write a uid-agnostic literal — e.g. `${XDG_RUNTIME_DIR}/agent-webhook-bridge-channel-<name>.sock` — instead of pinning `/run/user/<uid>/…` (which silently breaks live-wake when the install is restored on a host where the uid changed). `${XDG_RUNTIME_DIR}` resolves to `$XDG_RUNTIME_DIR`, or `/run/user/<uid>` when unset (PHP-FPM usually doesn't inherit it). An unresolvable token throws. `bridge:check` warns if the resolved parent dir is missing/non-writable. |
| `url` | loopback http URL | `null` | e.g. an SSH-tunnel endpoint. Non-empty, no whitespace (else throws); the loopback/SSRF check is the handler's. `bridge:check` **liveness-probes** the `host:port` (a TCP connect — reaches the remote connector through the reverse tunnel) and surfaces an HTTP `.FAILED` marker best-effort when run on the agent host (DL-156). |
| `route_intents` | bool | `false` | Dispatcher auto-pushes every staged intent here. **Requires** `socket` or `url` (else throws). The shipped hand-emitting classifiers (`EventDrivenClassifier`, `CoordinationClassifier`) are safe on either setting: both route their `channel_push` through the guarded `InboxOnlyClassifier::wakePush()`, which suppresses the hand-emit when `route_intents:true` (DL-191, DL-208), so the routed push is the single wake. Only a **raw custom classifier** that emits `channel_push` *without* `wakePush()` double-pushes here — pair those with `route_intents:false`. |
| `auth.token_path` | abs path to a file | `null` | Bearer token for the **HTTP transport** (`url`). **Rejected at load unless `url` is set** (DL-008). Read fail-closed at push: must be `0600`, non-empty (else the push errors, inbox backstops it). |

### `surface:` (optional)
| Key | Type | Default | Notes |
|---|---|---|---|
| `silent_drop_warnings` | bool | `true` | Warn when a `channel_push` target has no paired Intent. Non-bool throws. |

### `api:` (optional) — per-agent token-path override
| Key | Type | Default | Notes |
|---|---|---|---|
| `<provider>.token_path` | abs path | convention `<secret_dir>/<provider>/token` | Override only when the agent authenticates as a distinct account. |

Unknown top-level keys (outside `identity`/`subscriptions`/`echo_suppression`/`surface`/`classifier`/`channel`/`api`) are warned, not fatal.

## 3. Secrets, tokens & state files

| Path | Mode | Notes |
|---|---|---|
| `<secret_dir>/<provider>/webhook-secret-scope-<scope>` | `0600` | Per-(provider,scope) HMAC secret. Group/world-readable → receiver **500 `secret_perms_insecure`** (DL-010). `<scope>`'s `/` is `%2F`-encoded. |
| `<secret_dir>/<provider>/token` | `0600` | Per-provider API token used **only** by `bridge:provision`. Insecure perms → provision **fails** (DL-010). `bridge:check` *warns* (not fails) if it's absent/unreadable ("will SKIP `<provider>` scopes") — **that warning is only actionable if you (re-)run `bridge:provision`.** On a finished install where provisioning was a one-time admin action and the token was intentionally removed, the warning is expected and safe to ignore (the running receiver never reads this token). |
| `<config_dir>/shared-identities.json` | `0600` | Optional. `{ "shared_identities": [ {github_user_id, github_login?, agents: [...]} ] }` — one declaration for a GitHub account shared by multiple agents (DL-002). Resolves to `Actor.name = null` so a classifier re-attributes. |
| `<config_dir>/writeback.json` | `0600` | Optional (absent ⇒ writeback off, DL-009/019). `{ "identity_id": <kanban user>, "alert_channel": {<socket\|url>}, "mappings": { "owner/repo": { "board_id": N, "stages": { "opened"\|"merged"\|"merged_to_main"\|"closed_unmerged": <stage_id> }, "create_dependabot_cards": <bool, default false>, "create_coord_cards": <bool, default false>, "coord_card_stage_id": <id, required-when-create_coord_cards/move_coord_cards>, "move_coord_cards": <bool, DL-204 fleet default: absent ⇒ on where coord_card_terminal_stage_id present, else inert; explicit false opts out>, "coord_card_terminal_stage_id": <id, required-when-move_coord_cards>, "swimlane_id": <id, optional>, "draft_overlay": <bool, default false> } } }`. Per-install policy → config dir, not tracked config. Malformed mapping → `bridge:check` **fails** / move handler 5xx. `identity_id` is auto-seeded into the global echo set so the writeback's own `card_updated` doesn't loop. `swimlane_id` (DL-027) pins **created** cards to a lane (create-only, never moves a card; strict-numeric — a non-numeric value fails closed; `bridge:check` warns if the lane isn't on the board). **`unpark_from_stages`** (DL-194, optional list<int>) auto-unparks a **pinned** card on a branch-cut `started` event *from these stages only* (the DL-178 reversal, scoped) — parsed strictly like `started_from_stages` and **must be disjoint from it** (an overlap fails the config closed; `bridge:check` reports it + warns on an id not on the board). **`hold_marker_tags`** (DL-194, optional list<string>, default `[]`) widens the auto-unpark override **alert** to an install-specific hold convention (e.g. `gate`); it can never narrow/suppress the alert for a PinGuard-pinned card; empty ⇒ the fail-safe alerts on every non-benign-draft unpark. **`draft_block_reason`** (DL-194, optional non-empty string, default `"PR is in draft"` = `KanbanBlockReasonHandler::MARKER`) names the benign automated-draft `block_reason` sentinel so a draft-park doesn't alert on unpark; a present-but-empty value fails closed. **`revive_on_reopen`** (DL-195, optional bool, default false) revives a card parked in the mapped **`closed_unmerged`** (Won't-Do) stage back to the **`opened`** stage when its PR is **reopened** — the classifier emits a distinct `reopened` outcome that reuses `stages.opened` (no `stages.reopened` key); scoped to the abandon stage (terminal-safe), overrides a pin with a `writeback_revived_on_reopen` alert, redelivery-safe; `bridge:check` warns if `stages.opened`/`stages.closed_unmerged` is missing. Absent/false ⇒ `reopened` stays the `opened` outcome (byte-identical). See `docs/writeback.md`. **`create_coord_cards`** (DL-198, optional bool, default false) creates a tracking card in **real time** for a coordination issue opened/reopened with a recognized `[BRIEF]`/`[ANNOUNCE]`/`[QUERY]`/`[REVIEW]`/`[TASK]` title, keyed (idempotent, create-only) on the `id:<sid>` tag; requires **`coord_card_stage_id`** (the stage the card lands in — **required-when-create_coord_cards**, fails the config closed at load if absent, strict-numeric) and requires the `coord-card-create` `CoordinationClassifier` family enabled on the serving agent + a non-null `identity_id` (the echo-gate; `bridge:check` warns when unset). `bridge:check` warns if `coord_card_stage_id` isn't on the board. Absent ⇒ byte-identical. See `docs/writeback.md`. **`move_coord_cards`** (DL-200; a guarded **fleet default** since DL-204/#4357) moves that same tracking card in **real time** when its coordination issue **closes** (→ **`coord_card_terminal_stage_id`**) or **reopens** (→ back to `coord_card_stage_id`, the revive target). **Separately opt-in — it does NOT ride `create_coord_cards`.** Requires the `coord-card-move` `CoordinationClassifier` family enabled, plus BOTH **`coord_card_terminal_stage_id`** (**required-when-move_coord_cards**, strict-numeric, and **must differ from `coord_card_stage_id`**) and `coord_card_stage_id` (the revive target — absent, the leg half-works: closes land, reopens silently no-op); each fails the config **closed at load**. Correlated by the same `id:<sid>` tag — nothing carrying the tag ⇒ nothing moves (this leg never creates). A **revive is fail-closed on provenance**: only a **service-set** terminal (`last_stage_move.actor_type === "service"`, an allow-list) is reversed, and only from that terminal — a human's closure intent, an unknown/absent actor, or a card someone has moved on is never touched. A **close** overrides a human's user-lane placement (closure wins). `bridge:check` warns if `coord_card_terminal_stage_id` isn't on the board, and additionally performs the **cross-config compare** (see `COORD_CONFIG` in §1): it resolves the coordination config's terminal for that board and reports **agree / DISAGREE / CANNOT-VERIFY** — never fails (the compare + the terminal-agreement lines are gated on the `coord-card-move` family being enabled, so they never describe a leg that cannot fire). **Fleet default (DL-204):** an absent key resolves **ON where `coord_card_terminal_stage_id` is present** (its presence is the "move configured" signal), **inert (byte-identical) where absent**; an explicit `false` opts out. The leg fires only where the `coord-card-move` family is ALSO enabled — `bridge:check` nudges a family-enabled install whose terminal (hence the leg) is unset (config-only, scoped so a pure PR-writeback install stays quiet). A partial default-on config (terminal present, revive stage missing/equal) fails **closed at load**. See `docs/writeback.md`. **`card_id_tag_template`** (DL-206, optional non-empty string) stamps a rendered **`id:` provenance tag** on every dependabot card the bridge **creates** (`create_dependabot_cards`), so a tag-keyed Shipped→Released promoter can find them (the bridge otherwise mints dependabot cards without the `id:` tag its impl-created siblings carry). Free-form per-tenant grammar; placeholders **`{n}`/`{pr_number}`** = PR number, **`{repo}`** = repo NAME (last path segment). E.g. `"id:DEV-pr-{n}"` → `id:DEV-pr-166`, `"id:dep:{repo}#{n}"` → `id:dep:magento#166`. Absent/null ⇒ no tag (byte-identical); a present-but-empty or non-string value **fails closed at load**. Prepended to the `dependencies`/`triaged` tags; unknown placeholders left literal. See `docs/writeback.md`. **`draft_overlay`** (DL-193, optional bool, default false) mirrors a PR's **draft** state onto the correlated card's **`block_reason`** field — overlay ONLY, no stage/column move. `converted_to_draft` / opened-as-draft **SET** the marker `"PR is in draft"` add-if-missing (only when `block_reason` is empty — a human reason is never stomped); `ready_for_review` **CLEARS** it clear-if-ours (only when the current value is exactly that marker). Setting it pins the card (DL-178). Absent/false ⇒ those actions are ignored (byte-identical). Not a stage-mapped outcome — `stages` is untouched. See `docs/writeback.md`. **`promote_on_release`** (DL-207, optional bool, default false) promotes **Shipped** cards to **Released** on a release merge to `main`: on the `merged_to_main` event the bridge scans the mapped board and moves every card at **`stages.merged`** (Shipped, tooling-managed by a PR reference, not pinned) whose merged PR's commit is now reachable from `main` to **`stages.merged_to_main`** (Released) — the steady-state Shipped→Released transition a card-first board otherwise never gets (a base=dev card fires no `merged_to_main` of its own; `bridge:reconcile` defers this transition to "the promote workflow", which this IS). Reuses the existing `stages.merged`/`stages.merged_to_main`; **both are required** when on (**fails closed at load** if either is missing). Promotion keys on **POSITIVE reachability** — `compare(merge_commit_sha...main).status ∈ {ahead, identical}`, gated on the PR being `merged` — not a dev-only proxy. **Precondition:** the release→`main` merge must preserve dev's commit shas (a **merge-commit / fast-forward**, this fleet's method); a squash/rebase RELEASE merge promotes nothing (unguardable — a doc precondition). **First runtime GitHub-read dependency:** the leg runs in the FPM receiver, where `GH_TOKEN` is absent and the credential-store helper is CLI-only (DL-184), so it needs a **placed `<secret_dir>/github/token`** (or `providers.github.token_path`) — unlike `bridge:reconcile` (CLI). Unresolved token / same-column stages ⇒ inert; `bridge:check` **warns** on both (there is no reconcile backstop for this transition). Absent/false ⇒ byte-identical. See `docs/writeback.md`. **`alert_channel`** (FR-4, optional) emits a loud per-event signal on a permanent move-failure, in addition to the log: `{ "socket": "/abs/path" }` OR `{ "url": "http://127.0.0.1:PORT/", "auth": { "token_path": "/abs/path" } }` (mutually-exclusive socket/url, loopback-only url). Opt-in, best-effort, deduped per `(repo, outcome, reason)`; a malformed alert_channel **warns** (never fails the config) — see `docs/writeback.md`. |
