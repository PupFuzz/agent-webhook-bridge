# Config schema (current-state reference)

The **what**: every configuration key the bridge reads, its type, default, and whether a bad value **fails closed** (throws/5xx/non-zero) or **warns**. The **why** of each choice lives in [`../CLAUDE_DECISIONS.md`](../CLAUDE_DECISIONS.md) (the `DL-NNN` log); this file is the flat reference so you don't have to reconstruct the schema by reading DL-002 + 005 + 006 + 007 + 008 + 010 + 011 + 014 as deltas. (B-11.)

There are three configuration surfaces:

1. **Per-install** â€” Laravel `.env` â†’ `config/bridge.php`. Same for every agent on one install.
2. **Per-agent** â€” `<config_dir>/<agent>.yml`. One file per agent; the **filename is the agent name**.
3. **Secrets / state** â€” files under `<secret_dir>` / `<state_dir>`, not committed.

## 2. Per-agent YAML (`<config_dir>/<agent>.yml`)

The **filename** (minus `.yml`) is the agent's canonical name and its echo "self" â€” there is **no `identity.self`** (DL-007). Malformed YAML throws at load â†’ **5xx** (fail-closed; kanban-board holds + redelivers). `chmod 600` the file.

### `identity:` (optional mapping) â€” the agent's own immutable upstream ids
| Key | Type | Default | Notes |
|---|---|---|---|
| `kanban_user_id` | int | `null` | The agent's kanban `user_id`. Auto-seeded into self echo-suppression. |
| `github_user_id` | int | `null` | The agent's immutable GitHub `sender.id`. Auto-seeded into self echo. |
| `github_login` | string | `null` | **Display-only label** â€” never a matching key (renameable; DL-002). A stale value logs a one-line drift warning. |

Recognition keys on the numeric ids only; the agent never hand-lists its own id for echo (auto-seeded, DL-007). Modelled by `IdentityConfig` (DL-017).

### `subscriptions:` (expected list â€” an agent with none receives nothing) â€” which provider scopes this agent receives
Each entry:
| Key | Type | Default | Notes |
|---|---|---|---|
| `provider` | `kanban` \| `github` | â€” | Must have an adapter (`bridge:check` **fails** on an unknown provider, DL-015). |
| `scopes` | list | â€” | kanban board ids (`[5]`) / GitHub `org/repo` slugs. A scope failing `ScopeId` validation is rejected (it's the path-traversal boundary). |
| `event_filter` | list | `[]` (all) | e.g. `["task.*","comment.*"]`. |

Non-array `subscriptions`, or an entry that isn't a mapping, throws at load. Only `kanban` is API-provisionable (`bridge:provision` skips github with a non-zero exit).

### `echo_suppression:` (optional) â€” OTHER agents only (self is auto-seeded)
| Key | Type | Default | Notes |
|---|---|---|---|
| `treat_as_echo` | list of agent names | `[]` | Other agents' writes to silently skip (by name). |
| `treat_as_echo_ids` | list of raw ids | `[]` | Extra raw provider ids to skip (rarely needed; self ids are auto-seeded). |
| `treat_as_signal` | list of agent names | `[]` | If non-empty, **only** these agents' writes are signal. **Fail-closed:** a name with no matching `<name>.yml` throws at **preflight (`bridge:check`) / first dispatch** (5xx), not at YAML load (DL-007 â€” `SignalAllowlist`). |

### `classifier:` (optional)
| Key | Type | Default | Notes |
|---|---|---|---|
| `class` | FQCN | `App\Bridge\Classifiers\InboxOnlyClassifier` | Must be resolvable + implement `Classifier`; `bridge:check` resolves it so a typo **fails** at preflight instead of a dispatch-time 5xx. |
| `config` | mapping | `{}` | Typed parameters for a **config-driven classifier** (`ClassifierConfig`), so a shared reference classifier is parameterized without hardcoding per-project specifics. Absent â‡’ all-defaults (back-compat). A present non-mapping â€” or a malformed key below â€” **throws** (fail-closed, like the rest of the config). Read by a classifier via `$ctx->agent->classifierConfig`. |

#### `classifier.config:` keys
| Key | Type | Default | Notes |
|---|---|---|---|
| `scope_author_map` | mapping | `{}` | `scope_id` (repo) â‡’ the sole author-agent on that repo â€” the primary attribution path for a **label-less** impl event (falls back to the `from:`/`FROM:` line). Keys lowercased for case-insensitive scope matching; a non-string key/value **throws**. |
| `families` | list<string> | `[]` (classifier default) | The event families a config-driven classifier runs (its config-gated pipeline). Lowercased; a blank entry **throws**. | `drop_title_all_of` | list<list<string>> | `[]` | **`coord-message` family.** Drop (no intent, no wake â€” before the recipient gate) a coordination subject whose **title** contains **every** substring of any one group (AND within a group, OR across groups) â€” for bookkeeping-title noise (e.g. `[["Rule E back-merge sync","paper-trail anchor"]]`). | `wake_membership` | list<string> | `[to_me, to_all, comment_to]` | **`coord-message` family (DL-190).** Which classes grant coord-message **live-wake** (channel_push): `to_me` (a `to:<me>` label), `to_all` (`to:all`), `from_me` (`from:<me>` â€” activity on a thread you opened), and `comment_to` (DL-192 â€” a body-`TO:<self>` grant, see below). **DEFAULT `[to_me, to_all, comment_to]`** â€” `comment_to` is **default-ON since DL-213** (the post-a-reply-and-wait flow â€” post to a counterparty, wait for their directed reply â€” is otherwise dark out of the box). **`from_me` stays OUT of the default** â€” it is the over-wake/self-wake knob: a coordinator opening many threads would else wake on every reply to them. Empty/absent â‡’ every failing workflow wakes (back-compat). **âš  This inverts the family's fail-loud default: a pattern matching NO real workflow â€” a typo, or a workflow later renamed â€” SILENTLY blackholes every CI-failure wake for the scope (no wake, no trace under the default `drop`). ### `board_tools:` (optional, DL-217) â€” the two-way board window (`board_my_cards` / `board_create_card`)
Absent â‡’ byte-identical no-op. A present block **defaults ON** (DL-217 v7): it does NOT need an `enabled` key. The classification decides throw-vs-suppress:

- **`enabled: true`** (explicit) â‡’ **fail-loud at load**: the scope fields + a resolvable bearer are required, and any malformation throws (an operator assertion that can't be satisfied is malformed config).
- **`enabled: false`** â‡’ a well-formed no-op (staging / opt-out); the fields below may be omitted. **Use this to stage a block silently** â€” a partial block on an HTTP agent with no `enabled` key otherwise suppresses AND `bridge:check` FAILs.
- **No `enabled` key** (the recommended default) â‡’ the block is attempted; if it can be satisfied (complete scope + a resolvable bearer) it comes **ON**, otherwise it **suppresses** (stays OFF, carries a reason `bridge:check` renders as a **FAIL** â€” never a load throw, so one under-configured agent can't 5xx the fleet).
- A **non-mapping** block or a **non-bool `enabled`** â‡’ suppressed (never enabled). `enabled` accepts ONLY `true`/`false`: `symfony/yaml` does **not** booleanize `yes`/`no`/`on`/`1` (those are strings/ints), so they suppress with a message naming the cure.

See [`docs/board-tools.md`](board-tools.md).
| Key | Type | Default | Notes |
|---|---|---|---|
| `enabled` | bool | *(absent â‡’ default-ON where satisfiable)* | Only `true`/`false`. `true` â‡’ fail-loud; `false` â‡’ well-formed no-op (may omit the fields below); absent â‡’ default-ON-where-satisfiable-else-suppress; non-bool (incl. bare `enabled:` â†’ null) â‡’ suppressed. |
| `transport` | `http` \| `ssh` | `ssh` **(since v0.68.0 / DL-225; was `http`)** | **Which front door authenticates the call (card 4952).** `http` â‡’ the loopback `POST /agent-tools/call` resolves the agent by **bearer** (below). `ssh` â‡’ the SSH-forced-command `bridge:tools-call` resolves the agent by the pinned `--agent` and carries **NO bearer** â€” so an ssh block that also sets `auth` is contradictory and fails (explicit â‡’ throws; default â‡’ suppresses). `ssh` lifts ONLY the bearer requirement â€” `board_id`/`swimlane_id`/`create_stage_id` stay required and fail-loud; a bad value THROWS. **âš  The default flipped `http` â†’ `ssh` in v0.68.0 (DL-225): an unset `transport` now reads as `ssh`. A block relying on the old implicit `http` default must set `transport: http` explicitly before upgrading to keep the loopback path â€” post-upgrade the bearer stops resolving over the HTTP door and the call fails closed (401). `bridge:check` emits a pre-upgrade advisory for an agent that landed on `ssh` via the default with no completed ssh setup.** See [`docs/multi-host.md`](multi-host.md#board-tools-two-way-ssh-forced-command-transport-card-4952). A bad `transport` value throws (explicit) / suppresses (default), never fails open. |
| `auth.token_path` | abs path to a file | â€” (deprecation **alias**; else the channel token) | The per-agent Bearer the channel server presents. **HTTP transport only** â€” under `transport: ssh` any `auth` key is a contradiction (fails closed). **By default the bearer is the agent's channel token** (`channel.auth.token_path`) â€” no new credential, no mint.
